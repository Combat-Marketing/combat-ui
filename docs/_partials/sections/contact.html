Contact

Contact cards, location cards, and a contact form layout. Each is CSS-only and works from CMS-generated HTML — no JavaScript required. Contact and location cards share the .cui-contact-methods list, so a phone number reads the same whether it sits on a support card or beside an office address. Use real <a href="mailto:"> / <a href="tel:"> links and an <address> element so search engines and assistive tech pick the details up directly.

Drop several contact cards into a .cui-card-grid for a contacts directory, and put a cui-map, <iframe>, or photo in a location card's media panel. The contact form styles layout only — its controls reuse the shared .cui-input / .cui-textarea / .cui-field-label classes, so it works as a plain <form> or wrapped in a cui-form for validation.

API reference
Class Purpose
.cui-contact-card Single point of contact. Carry data-variant="flat | borderless | inverse" for chrome, data-align="center" to center content. Children: .cui-contact-card-eyebrow, .cui-contact-card-title, .cui-contact-card-lead, .cui-contact-card-actions.
.cui-contact-methods Label / value list shared by both cards. Render as <ul> of .cui-contact-method rows (or a <dl>). data-layout="inline" lays the label beside the value. Children: .cui-contact-method-label, .cui-contact-method-value.
.cui-location-card Office / venue card: media panel plus body. data-orient="stacked" pins the media on top; data-variant="flat | borderless" for chrome. Children: .cui-location-card-media, .cui-location-card-body, .cui-location-card-name, .cui-location-card-address (on <address>), .cui-location-card-actions.
Contact form No dedicated class — compose cui-form (validation / submit), cui-field (labels / help / errors), and the .cui-input family, laid out with .cui-stack, .cui-grid (data-columns for side-by-side fields), and .cui-cluster (actions). See the Forms page.
